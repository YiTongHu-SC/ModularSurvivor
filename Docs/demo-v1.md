# 里程碑 1（第 1–2 周）：核心 2D 战斗循环雏形

## 目标 Demo 能展示什么？

- 一个简单平面关卡（可以是纯色平面），有：
- 一个玩家单位（小方块/占位模型）。
- 敌人从四面刷出来，向玩家靠近。
- 玩家自动攻击（射线或弹道）敌人，打死一批敌人，过一波。
- 没有背包、模块，纯粹是“幸存者式自动战斗核心”。

## 技术与设计重点

- 项目基础架构
    - 搭好基础目录结构和命名约定：
    - Core/（通用工具、2D 数学）。
    - Combat/（战斗系统）。
    - Waves/（刷怪系统）。
    - Backpack/（背包系统，先空着或最小骨架）。
    - 确定逻辑 2D 坐标系与 Unity 3D 映射方案（对齐你之前讨论的“2D 逻辑、3D 表现”）。
- 战斗核心（纯 2D 逻辑服务 + 最简单 View）
    - 纯 C# 战斗服务类（不继承 MonoBehaviour），如：
        - UnitData（位置、生命、攻击间隔、攻击范围等）。
        - CombatSystem：自动攻击、伤害结算。
        - WaveSystem：定时刷怪（直线靠近玩家即可）。
    - 简单的 UnitView：
        - 通过 unitId 绑定逻辑对象。
        - 每帧把 2D 位置映射到 3D Transform.position。
- 演示用 UI
    - 在屏幕上简单显示：
    - 当前波数、剩余敌人数量。
- 展示角度：
    - 强调“战斗逻辑完全在 2D 上跑，3D 只是壳”。
    - 强调代码可读性和未来可扩展性（Combat/Wave 独立服务）。
