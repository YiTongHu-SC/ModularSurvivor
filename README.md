# Modular Survivor

Modular Survivor 是一款使用 Unity 开发的 3D 低多边形（Low-Poly）“幸存者类”自动战斗游戏原型。  
核心特色是一个 **可拼装的模块化背包系统**：玩家通过在网格背包中摆放和连接不同模块，构建出多样化的战斗 Build。

> 项目目前处于原型 / 开发中阶段，未来计划以开源形式发布。

---

## 核心玩法概述

- **自动战斗（Survivor-Like）**
    - 角色在关卡中自动攻击接近的敌人。
    - 敌人分波次不断刷出，难度逐渐提升。
    - 玩家主要关注“构建与决策”，而不是手动操作战斗。

- **模块化背包拼装**
    - 背包是一个网格（Grid），可以放置各种形状的模块（Module）。
    - 模块拥有连接点（Connector），通过相互连接组成“网络”。
    - 模块网络会影响角色的：
        - 基础属性（攻击力、攻速、范围、移动速度等）
        - 自动攻击方式和技能类型（如多向射击、穿透、爆炸等）

- **循环核心：Build 驱动的 Roguelite 体验**
    1. 进入关卡，自动战斗一波敌人。
    2. 通关后，从若干模块奖励中选择一件或多件。
    3. 进入背包界面，拖拽/旋转模块，尝试优化整体连接与效果。
    4. 重新计算属性和技能，进入下一波更高难度的战斗。
    5. 重复上述循环，体验不同构筑路线与组合策略。

## 技术与架构要点

- **2D 逻辑 + 3D 表现**
    - 战斗判定（移动、射线、范围伤害）在 2D 平面上进行，提升可控性与可调试性。
    - 角色、敌人、弹道和场景使用 3D Low-Poly 模型进行展示。
    - 实现类似 Dota2：逻辑是平面化，但视觉是全 3D 的效果。

- **模块化系统设计（规划中/建设中）**
    - `Grid / Cell / Module / Connector` 清晰分层：
        - 网格与格子负责空间布局。
        - 模块数据通过 ScriptableObject 定义形状、连接点和基础效果。
        - 运行时解析模块连接，构建“模块网络”。
    - 提供可扩展的属性/技能结算管线：
        - 支持按模块网络组合生成不同武器/效果。
        - 设计为数据驱动，方便扩展和调平衡。

- **系统分层（示例方向）**
    - **逻辑服务（纯 C#）：**
        - `GridSystem`、`ConnectionResolver`、`StatCalculator`、`CombatSystem`、`WaveSystem` 等。
        - 不依赖 MonoBehaviour，便于测试和重用。
    - **Unity 表现层：**
        - `UnitView` / `ProjectileView` / `BackpackUI` 等 MonoBehaviour。
        - 将逻辑层输出（位置、属性、技能）同步到 3D 场景与 UI。

## 当前状态

- 项目处于 **个人开发中**，采用渐进式迭代：
    - 优先实现：基础战斗循环 → 简单模块系统 → 完整“战斗-奖励-背包-下一波”流程。
    - 随后逐步增强：模块连接规则、模块网络效果、多样敌人和技能表现。
- 代码结构与系统设计会持续重构和整理，以适合作为作品集/技术示例使用。
