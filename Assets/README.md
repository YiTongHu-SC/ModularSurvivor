﻿# ModularSurvivor - 资源总览

## 概述
模块化幸存者类游戏，采用"2D逻辑，3D表现"的架构设计。项目处于Demo-V1开发阶段，已实现核心战斗系统、输入管理、波次生成等功能。

## 目录结构

### 核心代码 (Scripts/)
- **Core/**：核心系统 - 输入管理、坐标系统、单位管理、事件系统
- **Combat/**：战斗系统 - 技能、移动、伤害计算、Buff系统
- **Waves/**：波次系统 - 敌人生成和波次管理
- **GameLoop/**：游戏循环 - 游戏管理、调试工具
- **UI/**：用户界面 - 游戏UI和菜单界面
- **Utils/**：工具类 - 数学运算、扩展方法

### 游戏资源
- **Prefabs/**：游戏对象预制体
- **Materials/**：材质资源
- **Models/**：3D模型文件
- **Sounds/**：音频文件
- **Sprites/**：2D图片资源
- **Configs/**：配置数据文件

### Unity设置
- **Scenes/**：场景文件
- **Settings/**：项目设置配置
- **Resources/**：运行时动态加载资源
- **Plugins/**：第三方插件和库

## 已实现功能

### 核心特性
- **统一输入系统**：支持键盘、手柄、移动端的多平台输入
- **2D逻辑坐标**：游戏逻辑在2D平面运行，3D仅用于表现
- **模块化架构**：各系统独立开发，通过事件通信
- **配置驱动**：游戏数值通过ScriptableObject配置

### 战斗系统
- **自动战斗**：类似幸存者游戏的自动攻击机制
- **技能系统**：支持激光、追踪等多种攻击类型
- **Buff系统**：支持速度、攻击力、DOT/HOT等6种效果类型
- **移动系统**：多种移动策略和路径计算

### 敌人系统
- **波次生成**：可配置的敌人波次系统
- **智能AI**：多种AI行为策略
- **动态难度**：随时间递增的游戏难度

## 架构特色

- **逻辑表现分离**：核心逻辑使用纯C#类，不依赖Unity MonoBehaviour
- **事件驱动**：模块间通过事件系统进行松耦合通信
- **可测试性**：业务逻辑支持单元测试
- **扩展性**：易于添加新功能和内容

## 开发指南

1. 新功能开发优先在相应模块的Scripts/目录下实现
2. 遵循现有的命名约定和代码结构
3. 使用README.md文档记录模块功能和设计思路
4. 保持逻辑与表现的分离原则
