# Scripts - 项目代码结构

## 概述
本目录包含项目的所有C#脚本，采用模块化架构设计，分离逻辑与表现层。

## 架构原则
- **2D逻辑，3D表现**：所有游戏逻辑在2D坐标系中运行，Unity的3D系统仅用于视觉表现
- **服务导向**：核心系统设计为不继承MonoBehaviour的纯C#服务类
- **模块化**：各功能模块相互独立，通过事件系统和接口进行通信

## 模块结构
- `Core/` - 核心系统和通用功能
- `Combat/` - 战斗系统
- `Waves/` - 敌人波次系统
- `Backpack/` - 背包和装备系统（预留）
- `UI/` - 用户界面
- `Utils/` - 工具类和扩展方法

## 开发指导
1. 业务逻辑优先考虑放入对应的服务类
2. MonoBehaviour主要用于View层和Unity生命周期管理
3. 使用事件系统进行模块间通信
4. 保持代码的可测试性和可读性
